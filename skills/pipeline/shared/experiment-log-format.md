# Experiment Log Format

**Two log types:** (1) Per-experiment structured log at `experiments/logs/<exp_id>.md` — written by exec sub-agents. (2) Aggregate event log at `progress/lab.log` — written by Lab Agent during polling.

Every experiment — pilots (Phase 4), method iterations (Phase 5), full experiments (Phase 8) — gets a log at `experiments/logs/<experiment_name>.md`.

**Create the file BEFORE launching. Fill in result fields AFTER completion.**

Filename must match the experiment `id` in `dispatch/state.json`.

## Traceability: three anchors

Every experiment is fully traceable via three linked records:

| Record | What it captures | How to find it |
|--------|-----------------|----------------|
| **git tag** `exp/<project-slug>/<YYYYMMDD-HHMM>` | Exact code version | `git show <tag>` |
| **wandb run** | Metrics, config, hostname, GPU, git commit (auto) | wandb dashboard or `run.url` |
| **this log file** | Why the experiment was run, expected outcome, human observations | `experiments/logs/<id>.md` |

### Git tag procedure (before launching each experiment)

```bash
git add experiments/logs/<exp_id>.md experiments/scripts/ dispatch/state.json
git commit -m "exp: queue <exp_id>" || true   # commit any pending changes
git tag exp/<project-slug>/<YYYYMMDD-HHMM>
git push origin --tags
```

`project-slug` is the kebab-case project identifier from `config/config.md` (same as `project_name`).

### wandb init (required in every experiment script)

```python
import os, wandb, torch

run = wandb.init(
    project="<project-name>",        # same as git repo name
    name="<exp_id>",                 # e.g. "exp1_cifar10c_main"
    tags=["phase8", "round-1"],      # phase + idea round
    config={
        "dataset": ...,
        "model": ...,
        "lr": ...,
        # all hyperparams — wandb captures git commit + hostname automatically
        # Environment fingerprint ("扣子") — cross-machine traceability
        "env/conda":        os.environ.get("CONDA_DEFAULT_ENV", "unknown"),
        "env/cuda_version": torch.version.cuda or "cpu",
        "env/torch":        torch.__version__,
        "env/gpu_name":     torch.cuda.get_device_name(0) if torch.cuda.is_available() else "cpu",
        "env/hostname":     os.uname().nodename,
    }
)
# ... training loop ...
wandb.log({"ece": ece, "acc": acc, "loss": loss})
wandb.finish()
```

wandb automatically records: `git.commit`, `host`, `gpu`, `pip` packages, OS info. The `env/*` fields above add the **environment fingerprint** — when comparing runs across machines, check that `env/conda`, `env/cuda_version`, and `env/torch` match. Discrepancies explain result gaps before you blame the algorithm.

## Log Template

```markdown
# Experiment Log: [experiment_name]

**Phase**: [Phase N — Pilot / Method Iteration N / Full Experiment]
**Idea Round**: [N]
**Git tag**: exp/<project-slug>/<YYYYMMDD-HHMM>
**wandb run**: [URL from wandb.run.url — fill in after launch]
**Date started**: ISO 8601 with seconds: `YYYY-MM-DDTHH:MM:SSZ` (UTC). Example: `2026-04-05T14:30:45Z`
**Date finished**: ISO 8601 with seconds: `YYYY-MM-DDTHH:MM:SSZ` (UTC). Example: `2026-04-05T14:30:45Z`

## Motivation
[Why is this experiment being run? What specific hypothesis does it test?]

## Setup
- **Script**: `experiments/scripts/[script_name].py`
- **Dataset**: [dataset name and split]
- **Key hyperparameters**: [non-default settings]
- **Baseline / comparison**: [what this is compared against]

## Expected Outcome
[What result would confirm or refute the hypothesis?]

## Actual Result
| Metric | Value | vs. Baseline |
|--------|-------|-------------|
| [metric] | [value] | [+/- delta] |

## Observations & Notes
[Anomalies, OOM events, fixes applied, follow-up ideas]
```

## progress/lab.log (Lab Agent aggregate log)

Written by Lab Agent during Phase 4/8 polling — not by exec sub-agents. Provides a running timeline of all experiment events across a project in one place.

**Format:** one line per event:

```
[HH:MM:SS] [EXP_ID] <event>
```

Where `<event>` is one of:

| Event | Meaning |
|-------|---------|
| `started` | Exec agent launched the experiment |
| `done (Δ=+2.3pp)` | Experiment finished; delta vs. baseline |
| `failed (OOM)` | Experiment failed with reason |
| `retrying as exp_001_r2` | Being re-queued under a new ID |
| `on_hold (timeout)` | Machine unreachable; held pending manual check |

**Example entries:**

```
[09:14:32] [exp_001] started
[09:47:11] [exp_001] done (Δ=+2.3pp)
[09:47:15] [exp_002] started
[10:03:58] [exp_002] failed (OOM)
[10:04:01] [exp_002] retrying as exp_002_r2
[10:04:03] [exp_002_r2] started
[11:22:44] [exp_003] on_hold (timeout)
```

## Rules

- Never skip the log, even for 5-minute pilot runs
- If re-run with different settings: new file (`exp1_cifar10c_main_v2.md`), new git tag, new wandb run
- Machine & hardware info is in the wandb run — no need to repeat it here

## all_results.csv Schema

**Location**: `experiments/results/all_results.csv`  
**Generated by**: `experiments/scripts/export_results.py` (Phase 8.5)  
**Purpose**: Single source of truth for all experiment results used in Phase 9 analysis and paper writing.

### Required Columns

| Column | Type | Example | Description |
|--------|------|---------|-------------|
| `exp_id` | str | `exp1_cifar10c_main_s0` | Experiment ID from dispatch/state.json |
| `method` | str | `TTAC` | Algorithm name (must match paper tables exactly) |
| `dataset` | str | `cifar10c` | Dataset name |
| `group` | str | `main` | One of: `main`, `baseline`, `ablation`, `analysis` |
| `metric` | str | `acc` | Metric name (e.g., `acc`, `ece`, `f1`) |
| `seed` | int | `0` | Random seed used |
| `value` | float | `92.3` | Metric value |
| `host` | str | `xuchang-lab1` | Machine hostname where the experiment ran |
| `gpu` | int | `1` | GPU index used (0-indexed) |
| `finished_at` | str | `2026-04-05T14:30:45Z` | ISO 8601 UTC timestamp when experiment completed. Written by the training script as the last step before exiting. Example code: `row['finished_at'] = datetime.datetime.utcnow().strftime('%Y-%m-%dT%H:%M:%SZ')` |

### Group Conventions

| Group value | Meaning | Used in |
|-------------|---------|---------|
| `main` | Proposed method (our approach) | Main results table |
| `baseline` | Published baselines from literature | Comparison rows |
| `ablation` | Ablated variants (components removed) | Ablation table |
| `analysis` | Diagnostic/visualization experiments | Appendix |

**Important**: The `group` column is case-sensitive. The early_stop_check.py and statistical tests filter by exact string match. Do NOT use custom group names.

> **Note:** `finished_at` must be written by the experiment training script itself, not by exec agent. If your script does not write this column, exec_local.md's CSV validation will fail.

### Validation

Before Phase 9 analysis, verify the CSV is complete:
```python
import pandas as pd
df = pd.read_csv("experiments/results/all_results.csv")
required = {"exp_id", "method", "dataset", "group", "metric", "seed", "value", "host", "gpu", "finished_at"}
assert required.issubset(df.columns), f"Missing columns: {required - set(df.columns)}"
assert df["value"].notna().all(), "NaN values in results — remove incomplete rows"
assert df["group"].isin(["main","baseline","ablation","analysis"]).all(), "Invalid group values"
```

## Sidecar Fields (dispatch/<EXP_ID>.status.json)

The following fields are stored in per-experiment sidecar files (NOT in all_results.csv):

| Field | Type | Description |
|-------|------|-------------|
| `hf_artifact_url` | str (optional) | HuggingFace artifact URL after checkpoint upload. Written by training script on successful upload. Empty string if upload did not occur. Used by exec_local.md to gate checkpoint cleanup. |
| `wandb_run_id` | str (optional) | Full wandb run URL (e.g. `https://wandb.ai/<entity>/<project>/runs/<id>`). Written by exec agent. |
| `pid` | int (optional) | Process ID of the running experiment. Written by exec agent, cleared on completion. |
| `retry_count` | int | Number of retry attempts so far. |
