快速开始

三步上手

无需安装额外依赖，服务器在 ~/.claude/skills/autoresearch-dashboard/。

Step 1 · 启动 Dashboard 服务器

bash 
$ pgrep -f "autoresearch-dashboard/server.py" > /dev/null || \
  nohup python3 ~/.claude/skills/autoresearch-dashboard/server.py > /tmp/rdb.log 2>&1 &
$ echo "Dashboard: http://$(hostname -I | awk '{print $1}'):8080"
# → Dashboard: http://10.165.232.227:8080

Step 2 · 在实验脚本中接入 tracker

python 
import sys, os
sys.path.insert(0, os.path.expanduser("~/.claude/skills/autoresearch-dashboard"))
import tracker

run = tracker.init(
    project="my-project",      # HOME 下的目录名
    name="exp1_cifar10c_main_s0",
    host="10.165.232.227",       # Dashboard 服务器 IP
    config={
        "method":  "our_method",
        "dataset": "cifar10c",
        "group":   "main",        # "main" / "baseline" / "ablation"
        "seed": 0,
        "lr": 0.001,
    },
)

# 训练循环中每 N 步自动上报
for step in range(10000):
    loss = train_step()
    run.log({"loss": loss, "step": step})

# 结束时上报 final metrics → 写入 all_results.csv
run.finish({"final_acc": 92.3, "final_ece": 4.1})
group 字段说明：必须使用规范值 "main"（我们的方法）/ "baseline"（对比基线）/ "ablation"（消融实验）/ "analysis"。这直接影响结果表的列排序和 early-stop 逻辑。

Step 3 · 启动 auto-research 流水线

Claude 会自动调用 auto-research skill，从 Phase 0 Setup 开始，带你走完整个流程。

Debate Agents

多 Agent 辩论机制

研究过程中的关键决策由多个 Claude agent 从不同角色出发辩论，避免单点偏见。

💡

Idea Debate Phase 2

6 位模拟评审者 + 1 位 AC，对提出的 idea 从创新性、实验可行性、相关工作等维度打分辩论。只有通过 acceptance gate 才能进入下一阶段。

🔬

Experiment Design Debate Phase 7

4 位 agent 分别扮演方法设计者、统计学家、实现工程师、批评者，确保实验设计覆盖所有 baselines 且统计上合理。

📊

Result Debate Phase 9

6 位分析师从 Skeptic（质疑）、Advocate（辩护）、Statistician（统计）等角度分析结果，输出 go/no-go 决策。

📝

Peer Review Simulation Phase 11

模拟 ICML/NeurIPS 审稿流程：6 位评审 + 1 位 AC，给出评分和 rebuttal 要点，在投稿前发现弱点。

实验控制

Early-Stop 自动取消机制

方法在前几个 seed 就表现很差时，自动取消剩余实验，节省 GPU 时间。

工作原理

  1. 每次实验完成后运行 early_stop_check.py
  2. 读取 all_results.csv，计算 our method vs best baseline 的 improvement
  3. 如果所有已完成实验的 improvement 均 < threshold（默认 0.5%）
  4. 将该 group 内所有 pending 实验状态改为 "cancelled"
  5. Supervisor 不再启动被取消的实验

dispatch/state.json 配置

json 
{
  "id": "exp1_cifar10c_main_s0",
  "group": "main_cifar10c",       // group key
  "early_stop_check_after": 1,  // 几个完成后检查
  "early_stop_threshold_pct": 0.5,
  "early_stop_metric": "acc",
  "status": "pending"
}
常见问题

Tips & FAQ

▸  如何中途恢复（Claude context 断了）？
按顺序读取：plan/TODO.md（找当前 phase）→ progress/progress.md（看决策日志）→ dispatch/state.json（检查是否有 zombie running 状态）。然后告诉 Claude 当前 phase，它会自动继续。
▸  results 表里看不到我的实验？
Results 表只显示 run.finish() 调用后的实验（已写入 all_results.csv）。正在运行中的实验会出现在 Runs 视图里，状态为 running。检查 config 中是否设置了 "method" 和 "dataset" 字段。
▸  显著性标记 * 没有显示？
需要先运行 Phase 9.0 统计检验脚本生成 experiments/results/significance_tests.csv。Dashboard 会显示黄色警告提示。运行后刷新页面或切换项目重新载入。
▸  如何只做实验不写论文？
在启动时告诉 Claude「不用写文章」/ 「只要验证算法」，或者在 Phase 0 选择 research-only 模式。Phase 9 go/no-go 通过后输出 report/research_report.md 而不是 LaTeX 论文。
▸  项目名有空格或特殊字符？
建议使用 kebab-case（my-project）或下划线（my_project）。Dashboard API 会自动 URL encode。项目目录直接在 HOME 下。
